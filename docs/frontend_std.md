  - 标题+副标题：明确主题与读者收益，含关键技术词。
  - 读者画像与前置：适合的水平、需要的基础（框架/语言版本/包管理器等）。
  - 背景与问题：场景/痛点/目标，示例输入输出，衡量成功的标准。
  - 核心概念：必要术语/模型/依赖，避免正文重复解释。
  - 环境与依赖：Node/浏览器版本、工具链（Vite/SvelteKit、Bun 等）、安装命令。
  - 实践步骤：分步骤代码片段（可复制），每步的预期结果/截图。
  - 可运行示例：最小可运行 demo（仓库/沙盒链接），如何启动、如何验证。
  - 解释与取舍：为何这样做，与替代方案对比（性能/可维护性/无障碍/可测试性）。
  - 常见坑与 FAQ：兼容性、性能、边界条件、安全/无障碍注意事项。
  - 测试与验证：手动验证清单、关键单测/e2e 示例。
  - 性能与可访问性：指标（如 CLS/LCP/FID），ARIA/键盘支持、色彩对比说明。
  - 最佳实践清单：总结性的要点/检查项。
  - 总结与下一步：复盘要点，给出可扩展方向或进阶阅读。
  - 参考与链接：官方文档、标准、相关库、示例仓库。
  - 元信息：阅读时长、标签/分类、SEO 关键词、更新时间。
  - 行动号召（CTA）：试用、提交 issue、订阅/评论、Star 仓库。
