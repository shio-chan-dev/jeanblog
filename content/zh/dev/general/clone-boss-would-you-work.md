## 副标题 / 摘要

这类问题考察的是自我认知与协作方式，而不是“愿不愿意”的简单答案。本文给出结构化分析。

## 目标读者

- 面试准备者
- 关注团队协作的工程师
- 想提升沟通能力的人

## 背景 / 动机

“克隆上司”是一个价值观问题，考察你如何处理相似性、冲突与反馈。  
好的回答强调“合作机制”而非情绪判断。

## 核心概念

- **自我认知**：理解自己的优势与盲点
- **角色边界**：上下级的职责分工
- **反馈机制**：避免同温层与盲区

## 实践指南 / 步骤

1. **先肯定协作价值**
2. **识别潜在盲区**（过度相似）
3. **提出机制**（多样性、外部反馈）
4. **强调目标一致性**

## 可运行示例

```python
# 用“协作矩阵”表达观点

def collaborate(similarity, feedback):
    return "healthy" if similarity < 0.8 or feedback else "risky"


if __name__ == "__main__":
    print(collaborate(0.9, True))
```

## 解释与原理

过度相似会放大盲区，但清晰边界与反馈机制能缓解风险。  
回答重点在“如何合作”，而不是“是否喜欢”。

## 常见问题与注意事项

1. **需要给明确答案吗？**  
   可以给，但要解释理由与机制。

2. **如何避免“讨好式回答”？**  
   强调实际合作方式与风险管理。

3. **是否要强调多样性？**  
   是的，多样性是团队健康的基础。

## 最佳实践与建议

- 用结构化方式回答开放题
- 说明风险与对策
- 强调以目标为导向

## 小结 / 结论

克隆上司问题考察的是你对协作机制的理解。  
关键不在“愿意”，而在“如何让合作有效”。
